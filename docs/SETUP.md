# Mailserver Setup Guide

Complete guide to deploying owney on a fresh VPS or bare metal, including domain setup and backup configuration.

## Prerequisites

- A domain you control (with DNS access)
- A VPS or bare metal server (2+ cores, 4GB RAM, 20GB disk minimum)
- SSH access to the server
- (Optional) AWS account for S3 backups

## Step 1: Domain DNS Setup

Before deploying owney, configure your domain's DNS records.

```bash
# For interactive manual setup:
./scripts/setup-dns.sh example.com manual

# For AWS Route53:
./scripts/setup-dns.sh example.com route53

# For Cloudflare:
./scripts/setup-dns.sh example.com cloudflare

# For BIND zone files:
./scripts/setup-dns.sh example.com bind
```

This generates the DNS records needed:
- **MX Record** (priority 10) → `mail.example.com`
- **A Record** → Your server's IP
- **SPF Record** → `v=spf1 ip4:<your-ip> ~all`
- **DMARC Policy** → `v=DMARC1; p=quarantine; rua=mailto:dmarc@example.com`
- **DKIM** → Generated by owneyd after first run

Wait 5-15 minutes for DNS propagation, then verify:

```bash
dig MX example.com +short
dig A mail.example.com +short
dig TXT example.com +short
```

## Step 2: Configure Mailserver

On your server:

```bash
# Generate example config
owneyd config example | sed 's/example.com/example.com/g' > /etc/owney.toml

# Edit with your domain and settings
nano /etc/owney.toml
```

Key config sections:
- `[server]` — Hostname, TLS cert paths
- `[storage]` — SQLite DB and blob storage paths
- `[backup]` — (Optional) Local or S3 backups
- `[smtp]` — Inbound/outbound ports
- `[api]` — JMAP API port

## Step 3: Initialize Database

```bash
owneyd --config /etc/owney.toml admin init
```

This creates the SQLite database and applies all schema migrations.

## Step 4: Create Admin Account

```bash
owneyd --config /etc/owney.toml admin create-account admin@example.com
```

Save the password securely. Use this to set up IMAP/JMAP clients.

## Step 5: Setup Backups (Recommended)

### Local Backups

```bash
mkdir -p /var/backups/owney
owneyd --config /etc/owney.toml backup create --output /var/backups/owney
```

Schedule daily backups with cron:

```cron
0 2 * * * /usr/local/bin/owneyd --config /etc/owney.toml backup create --output /var/backups/owney
```

### S3 Backups (AWS)

```bash
./scripts/setup-s3-backup.sh example.com

# Add to [backup] section of owney.toml:
[backup]
type = "s3"
region = "us-east-1"
bucket = "owney-backup-example-com"
access_key = "..."
secret_key = "..."
```

Test with:

```bash
owneyd --config /etc/owney.toml backup create
```

## Step 6: Start Mailserver

### Systemd Service

Create `/etc/systemd/system/owneyd.service`:

```ini
[Unit]
Description=Mailserver
After=network.target

[Service]
Type=simple
User=mail
ExecStart=/usr/local/bin/owneyd --config /etc/owney.toml serve
Restart=always
RestartSec=10

[Install]
WantedBy=multi-user.target
```

Then:

```bash
systemctl enable owneyd
systemctl start owneyd
systemctl status owneyd
```

### Docker

```bash
docker build -t owney .
docker run -d \
  --name owney \
  -v /etc/owney.toml:/etc/owney.toml:ro \
  -v /var/lib/owney:/var/lib/owney \
  -p 25:25 \
  -p 143:143 \
  -p 80:80 \
  -p 443:443 \
  owney serve --config /etc/owney.toml
```

## Step 7: Generate DKIM Key

Once owneyd is running:

```bash
owneyd --config /etc/owney.toml dkim generate example.com
```

This outputs a TXT record. Add it to your DNS:

```
_domainkey._report.example.com.  3600  IN  TXT  "v=DKIM1; k=rsa; p=MIGfMA0B..."
```

Verify DKIM is working:

```bash
dig +short _domainkey._report.example.com TXT
```

## Step 8: Health Checks

The doctor daemon runs automatically in the background, checking:
- DNS resolution (MX, DKIM, SPF, DMARC)
- TLS certificate expiry
- Queue health
- Database integrity

View health status:

```bash
owneyd --config /etc/owney.toml doctor
```

This publishes events to the event bus for monitoring.

## Step 9: Test Email Flow

### Send Test Email

From the server:

```bash
echo "Test body" | owneyd --config /etc/owney.toml smtp send \
  --from admin@example.com \
  --to recipient@gmail.com
```

### Receive Test Email

Connect an IMAP client (Thunderbird, Apple Mail, etc.) to your server:

```
IMAP Host: mail.example.com
Port: 143 (or 993 for TLS)
Username: admin@example.com
Password: <from create-account>
```

Or use JMAP API:

```bash
curl -u admin@example.com:password https://mail.example.com/jmap
```

## Step 10: Update & Maintenance

### Safe Updates

Download a new owneyd binary, then:

```bash
# Verify binary (get hash from GitHub Releases)
owneyd --config /etc/owney.toml update owneyd.new \
  --hash abc123def456... \
  --dry-run

# Apply (swaps binary atomically, auto-rollback on DB migration failure)
owneyd --config /etc/owney.toml update owneyd.new \
  --hash abc123def456...
```

### Restore from Backup

If disaster strikes:

```bash
# Stop owneyd
systemctl stop owneyd

# Restore from backup
owneyd --config /etc/owney.toml backup restore /path/to/backup.tar.zst

# Verify
owneyd --config /etc/owney.toml admin accounts

# Start
systemctl start owneyd
```

## Monitoring

Configure monitoring to watch:
- CPU/Memory/Disk usage
- SMTP delivery queue depth (query DB for `queue` table status)
- TLS certificate expiry (doctor daemon checks this)
- Event bus for `DoctorCheck` events

Example Prometheus queries:

```promql
# Queue size
SELECT COUNT(*) FROM queue WHERE status = 'queued'

# Delivery rate
SELECT COUNT(*) FROM queue WHERE status = 'delivered' AND updated_at > datetime('now', '-1 hour')

# DB size
SELECT page_count * page_size FROM pragma_page_count(), pragma_page_size()
```

## Troubleshooting

### No mail arriving

1. Check DNS: `dig MX example.com`
2. Check firewall: port 25 (SMTP) must be open
3. Review logs: `journalctl -u owneyd -f`
4. Check queue: `sqlite3 /var/lib/owney/mail.db "SELECT COUNT(*) FROM queue"`

### Slow delivery

1. Check SMTP peers are responsive: `doctor` output
2. Check disk I/O: `iostat 1 10`
3. Check queue processing: monitor background tasks

### Database corruption

1. Stop owneyd: `systemctl stop owneyd`
2. Restore backup: `backup restore /path/to/backup.tar.zst`
3. Verify: `admin accounts`
4. Restart: `systemctl start owneyd`

### Certificate renewal

Mailserverd reads TLS certs from disk at startup. To renew:

1. Update cert files (e.g., let's encrypt with certbot)
2. Restart owneyd: `systemctl restart owneyd`
